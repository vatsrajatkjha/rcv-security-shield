# Security Policy

## Supported Versions

Security fixes are prioritized for the latest release line.

| Version | Supported |
|---|---|
| Latest stable | Yes |
| Older versions | Best effort |

## Reporting a Vulnerability

Please do **not** open public issues for vulnerabilities.

Instead, report privately to the maintainers with:

- affected version
- impact summary
- reproduction steps or proof of concept
- any suggested mitigation

If you do not have a private contact yet, open a minimal public issue requesting a private channel **without** disclosing details.

## Disclosure Process

- We acknowledge receipt as quickly as possible.
- We validate and triage severity.
- We work on a fix and release patch notes when available.
- We aim for responsible coordinated disclosure.
